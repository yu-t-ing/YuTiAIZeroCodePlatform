# 🎯 设计思路与设计重难点

## 一、项目背景与设计目标

### 1.1 项目背景
随着 AI 技术的快速发展，传统软件开发仍然存在较高的技术门槛。非技术人员想要实现一个简单的 Web 应用，需要学习 HTML、CSS、JavaScript 等多种技术，开发周期长、成本高。本项目旨在利用 AI 大语言模型的能力，让用户通过自然语言描述需求，自动生成完整的代码，实现真正的"零代码"开发。

### 1.2 设计目标
- **降低开发门槛**：用户无需编程基础，通过自然语言即可生成应用
- **提升开发效率**：从需求到应用的周期缩短至分钟级
- **保证代码质量**：生成的代码规范、可维护、可直接部署
- **良好的用户体验**：现代化的 UI 设计，实时展示生成过程
- **完善的系统功能**：用户管理、权限控制、应用管理等完整闭环

## 二、整体架构设计

### 2.1 技术架构选型

**后端技术栈选择理由：**
- **Spring Boot 3.5.4**：成熟的 Java 企业级框架，生态完善，便于快速开发
- **LangGraph4j**：创新的 AI 工作流编排框架，支持复杂的多步骤 AI 任务
- **Spring AI**：Spring 官方的 AI 集成框架，与 Spring Boot 无缝集成
- **MyBatis-Flex**：轻量级 ORM 框架，性能优异，使用简单
- **MySQL 8.0**：成熟的关系型数据库，支持 JSON 字段，适合存储对话历史

**前端技术栈选择理由：**
- **Vue 3.5.13**：采用 Composition API，代码组织更清晰，性能更好
- **Ant Design Vue**：企业级 UI 组件库，组件丰富，开箱即用
- **TypeScript**：提供类型安全，减少运行时错误
- **Vite**：极速的构建工具，开发体验优秀

## 三、核心功能模块设计

### 3.1 用户认证与权限模块

**设计思路：**
- 基于角色的访问控制（RBAC），分为普通用户和管理员
- 使用 Session 存储登录状态
- 密码使用 BCrypt 加密存储
- 自定义权限注解 `@AuthCheck`，简化权限校验

**数据表设计：**
```sql
CREATE TABLE user (
    id BIGINT PRIMARY KEY,
    userAccount VARCHAR(256) UNIQUE,  -- 账号
    userPassword VARCHAR(512),         -- 加密密码
    userName VARCHAR(256),             -- 昵称
    userAvatar TEXT,                   -- 头像（base64）
    userProfile VARCHAR(512),          -- 个人简介
    userRole VARCHAR(256) DEFAULT 'user', -- 角色
    createTime DATETIME,
    updateTime DATETIME,
    isDelete TINYINT DEFAULT 0
);
```

### 3.2 AI 代码生成模块

**设计思路：**
- 使用 LangGraph4j 构建状态机，管理复杂的多步骤 AI 任务
- 支持单页 HTML 和多文件 Vue 项目两种生成模式
- 通过 SSE（Server-Sent Events）实现实时流式输出
- 智能提示词工程，提升生成代码质量

**核心流程设计：**
```java
// AI 工作流状态定义
public class WorkflowContext {
    private String userPrompt;           // 用户原始需求
    private String enhancedPrompt;       // 增强后的提示词
    private String generatedCode;        // 生成的代码
    private QualityResult qualityCheck;  // 质量检查结果
    private String finalCode;           // 最终代码
}

// 工作流节点定义
State<WorkflowContext> workflow = new StateGraph<WorkflowContext>(State::new)
    .addNode("promptEnhancer", new PromptEnhancerNode())
    .addNode("codeGenerator", new CodeGeneratorNode())
    .addNode("qualityChecker", new CodeQualityCheckNode())
    .addEdge("promptEnhancer", "codeGenerator")
    .addEdge("codeGenerator", "qualityChecker");
```

### 3.3 对话历史管理模块

**设计思路：**
- 每个应用对应独立的对话上下文
- 使用 MySQL 存储完整对话历史
- 支持分页查询，优化大数据量性能
- 为 AI 提供上下文记忆，提升连续对话体验

**数据表设计：**
```sql
CREATE TABLE chat_history (
    id BIGINT PRIMARY KEY,
    appId BIGINT,                       -- 关联应用
    userId BIGINT,                      -- 用户ID
    message LONGTEXT,                   -- 消息内容（JSON格式）
    messageType VARCHAR(32),            -- 消息类型：user/ai
    createTime DATETIME,
    INDEX idx_appId_createTime (appId, createTime)
);
```

### 3.4 用户个人中心模块

**设计思路：**
- 支持头像上传（base64 格式存储）
- 实时表单验证
- 友好的 UI 反馈
- 数据安全校验（防止越权修改）

**实现要点：**
- 前端使用 FileReader 将图片转换为 base64
- 后端对图片大小和格式进行校验
- 使用 `TEXT` 类型存储 base64 字符串
- 更新成功后刷新 Session 中的用户信息

## 四、数据库设计

### 4.1 核心表设计

**用户表 (user)**
- 主键：自增 ID
- 唯一索引：userAccount
- 普通索引：userName（用于搜索）
- userAvatar 字段使用 TEXT 类型，支持存储 base64 图片

**应用表 (app)**
- 主键：自增 ID
- 外键：userId（关联用户）
- 唯一索引：deployKey（部署标识）
- 复合索引：appId + createTime（游标分页）

**对话历史表 (chat_history)**
- 主键：自增 ID
- 外键：appId、userId
- 复合索引：appId + createTime（提升查询性能）
- message 字段使用 LONGTEXT，支持存储大量内容

### 4.2 数据库优化策略

- **索引优化**：为高频查询字段添加索引
- **分页优化**：使用游标分页替代传统 offset
- **字段类型优化**：userAvatar 从 VARCHAR(1024) 改为 TEXT

## 六、设计重难点与解决方案

### 难点一：AI 代码生成的稳定性

**问题描述：**
- AI 模型输出格式不稳定，可能包含多余说明文字
- 生成的代码可能包含语法错误
- 长文本生成可能被截断

**解决方案：**

1. **提示词工程优化**
```java
String systemPrompt = "你是一个专业的代码生成助手。请严格按照以下格式输出：" +
    "```html\n" +
    "你的代码\n" +
    "```\n" +
    "不要包含任何其他说明文字。";
```

2. **输出清洗与解析**
```java
// 使用正则表达式提取代码块
Pattern pattern = Pattern.compile("```html\\n([\\s\\S]*?)\\n```");
Matcher matcher = pattern.matcher(aiResponse);
if (matcher.find()) {
    String code = matcher.group(1);
}
```

3. **质量检查机制**
- 检查代码完整性
- 验证 HTML 标签闭合
- 检查常见语法错误

### 难点二：SSE 流式输出的性能优化

**问题描述：**
- AI 生成的代码可能很长，每个 token 都触发一次 DOM 更新
- 频繁的 DOM 操作导致页面卡顿
- Markdown 渲染和代码高亮消耗大量资源

**解决方案：**

1. **节流机制**
```javascript
let fullContent = ''
let updateTimer = null

// 每 100ms 批量更新一次 UI
if (!updateTimer) {
  updateTimer = setTimeout(() => {
    updateUI()
    updateTimer = null
  }, 100)
}
```

2. **减少滚动频率**
```javascript
// 每 10 次更新才滚动一次
if (updateCount % 10 === 0) {
  scrollToBottom()
}
```

3. **延迟渲染复杂组件**
- 代码高亮在输出完成后执行
- 使用 Web Worker 处理 Markdown 解析

### 难点三：用户头像存储

**问题描述：**
- 图片转换为 base64 后数据量巨大
- 数据库字段长度限制
- 上传文件大小和格式验证

**解决方案：**

1. **数据库字段优化**
```sql
ALTER TABLE user MODIFY COLUMN userAvatar TEXT COMMENT '用户头像';
```

2. **前端图片验证**
```javascript
const handleBeforeUpload = (file: File) => {
  // 检查文件类型
  const isImage = file.type.startsWith('image/')

  // 检查文件大小（限制 2MB）
  const isLt2M = file.size / 1024 / 1024 < 2

  // 使用 FileReader 转换为 base64
  const reader = new FileReader()
  reader.onload = (e) => {
    tempAvatarUrl.value = e.target.result as string
  }
  reader.readAsDataURL(file)
}
```

3. **考虑未来优化**
- 可接入对象存储服务（如腾讯云 COS）
- 前端压缩图片后再上传

### 难点四：AI 工作流的状态管理

**问题描述：**
- 多步骤 AI 任务的状态同步复杂
- 某个步骤失败需要重试
- 需要支持动态路由决策

**解决方案：**

1. **使用 LangGraph4j 状态机**
```java
State<WorkflowContext> workflow = new StateGraph<WorkflowContext>(State::new)
    .addNode("promptEnhancer", new PromptEnhancerNode())
    .addNode("codeGenerator", new CodeGeneratorNode())
    .addNode("qualityChecker", new CodeQualityCheckNode())
    .addEdge("promptEnhancer", "codeGenerator")
    .addConditionalEdge(
        "qualityChecker",
        new RouterNode(),
        Map.of(
            "pass", "end",
            "fail", "codeGenerator"  // 失败则重新生成
        )
    );
```

2. **状态持久化**
- 将工作流状态保存到数据库
- 支持断点续传和失败重试

### 难点五：前端状态管理复杂度

**问题描述：**
- 用户信息、应用列表、对话历史需要全局共享
- 登录状态需要在多个组件中使用
- 异步数据加载状态管理

**解决方案：**

1. **使用 Pinia 状态管理**
```typescript
// stores/loginUser.ts
export const useLoginUserStore = defineStore('loginUser', {
  state: () => ({
    loginUser: {
      id: 0,
      userName: '未登录',
      userRole: UserRoleEnum.USER,
    }
  }),
  actions: {
    async fetchLoginUser() {
      // 获取登录用户信息
    }
  }
})
```

2. **路由守卫**
```typescript
router.beforeEach((to, from, next) => {
  const loginUserStore = useLoginUserStore()
  if (!loginUserStore.loginUser.id && to.path !== '/user/login') {
    next('/user/login')
  } else {
    next()
  }
})
```

### 难点六：并发安全与数据一致性

**问题描述：**
- 多个请求同时修改用户信息
- 应用生成时并发创建记录
- 对话历史的顺序性保证

**解决方案：**

1. **数据库乐观锁**
```java
@Version
private Integer version;
```

2. **唯一索引约束**
```sql
ALTER TABLE user ADD UNIQUE INDEX uk_userAccount (userAccount);
ALTER TABLE app ADD UNIQUE INDEX uk_deployKey (deployKey);
```

3. **事务管理**
```java
@Transactional(rollbackFor = Exception.class)
public Long createApp(AppAddRequest request) {
    // 创建应用
    // 保存初始对话
}
```
